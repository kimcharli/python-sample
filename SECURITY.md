# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

Please report security vulnerabilities by emailing the maintainer or creating a private security advisory on GitHub.

Do not create public issues for security vulnerabilities.

### What to include

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

### Response Timeline

- We will acknowledge your report within 48 hours
- We will provide a detailed response within 7 days
- We will work to fix confirmed vulnerabilities as quickly as possible

## Security Best Practices

This project follows security best practices:

- Dependencies are regularly updated using Dependabot
- Code is scanned using Bandit for security issues
- All code changes go through review process
- Automated testing includes security checks
